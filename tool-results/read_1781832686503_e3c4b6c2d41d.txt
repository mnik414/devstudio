     1→# DevStudio — Web Development Agency Portfolio Platform
     2→
     3→## Project Overview
     4→A premium web development agency portfolio platform. Originally requested as Laravel/PHP/Filament, delivered as the equivalent Next.js 16 + TypeScript + Prisma + shadcn/ui stack (per environment constraints). All requested features are implemented: Home, Portfolio (+detail), Case Studies (+detail), Blog (+detail), Contact, Estimate calculator, About, Admin panel, SEO (sitemap, robots, structured data), dark mode.
     5→
     6→## Tech Stack
     7→- Next.js 16 (App Router) + TypeScript
     8→- Prisma ORM + SQLite
     9→- Tailwind CSS 4 + shadcn/ui (New York)
    10→- Framer Motion (animations)
    11→- Zustand (view navigation) + TanStack Query (server state)
    12→- next-themes (dark mode)
    13→
    14→## Color System
    15→- Primary: #2563EB (blue)
    16→- Secondary: #0F172A (slate)
    17→- Accent: #14B8A6 (teal)
    18→- Background: white (light) / deep slate (dark)
    19→
    20→## Architecture (single-route SPA)
    21→- All content rendered on `/` (per environment rule)
    22→- Zustand `useNav` store switches between views: home | portfolio | case-studies | blog | contact | estimate | about | admin
    23→- Detail views (portfolio/blog/case-study) triggered via `openDetail(view, slug)`
    24→- API routes under `/api/*` for all data + admin CRUD
    25→
    26→## Database (Prisma models)
    27→Portfolio, PortfolioCategory, Technology, BlogPost, BlogCategory, Tag, CaseStudy, Testimonial, TeamMember, Service, Lead, ContactRequest, Faq, Setting. Seeded via `src/lib/seed.ts`.
    28→
    29→## Key Files
    30→- `src/app/page.tsx` — SPA shell (view switcher)
    31→- `src/lib/store.ts` — navigation state
    32→- `src/lib/hooks.ts` — data fetching hooks (TanStack Query)
    33→- `src/components/site/*` — navbar, footer, reveal, counter, section-heading, logo, theme-toggle, scroll-progress
    34→- `src/components/views/*` — all view components
    35→- `src/app/api/*` — API routes (portfolio, blog, case-studies, site, contact, estimate, admin, filters, blog-filters)
    36→
    37→## API Routes
    38→- GET /api/site — testimonials, team, services, faqs, settings
    39→- GET /api/portfolio?featured&category&tech&q&sort&limit
    40→- GET /api/portfolio/[slug] — detail + related + view increment
    41→- GET /api/filters — portfolio categories + technologies
    42→- GET /api/blog?category&tag&q&featured&limit
    43→- GET /api/blog/[slug] — detail + related
    44→- GET /api/blog-filters — blog categories + tags
    45→- GET /api/case-studies?featured
    46→- GET /api/case-studies/[slug]
    47→- POST /api/contact — store contact request (validated)
    48→- POST /api/estimate — calculate estimate + store lead
    49→- GET/POST/PATCH/DELETE /api/admin?model=... — CMS CRUD (X-Admin-Token auth)
